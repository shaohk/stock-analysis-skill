# Stock Analysis - Multi-Agent 股票分析框架

基于 Claude Code Agent Tool 的多智能体股票分析系统，通过 13 个专业 Subagent 协同工作，输出完整的投资决策报告。

## 项目背景

本项目参考 [TradingAgents-CN](https://github.com/chenweize/tradingagents-cn) 的多智能体协作思路，使用 Claude Code 的 prompt engineering 实现了完整的股票分析框架。

### 核心理念

将专业投资机构的研究流程拆解为多个角色，每个角色专注于特定领域，通过"分工-辩论-决策"的机制，最终形成客观、全面的投资建议。

## Agent 角色说明

### Phase 1: 分析师团队（并行执行）

| Agent | 职责 | 输出内容 |
|-------|------|----------|
| **技术分析师** | K线形态、均线系统、MACD/RSI/Bollinger等技术指标分析 | 技术面报告、买卖信号、支撑阻力位 |
| **新闻分析师** | 公司公告、行业动态、政策解读 | 新闻影响评估、事件驱动分析 |
| **市场分析师** | 市场结构、流动性、资金流向分析 | 市场环境评估、行业轮动分析 |
| **基本面分析师** | 财务报表、估值模型(PE/PEG/DCF)、竞争力分析 | 估值报告、成长性评估 |
| **情绪分析师** | 市场情绪、投资者行为分析 | 情绪指标、恐慌贪婪指数 |
| **中国市场分析师** | A股/港股特色分析、南向资金、政策影响 | 中国市场专项分析 |
| **社交媒体分析师** | 社交平台舆情、投资者讨论热度 | 舆情报告、热度趋势 |

### Phase 2: 多空研究员（对抗辩论）

| Agent | 立场 | 任务 |
|-------|------|------|
| **多头研究员** | 寻找买入理由 | 论证上涨逻辑、挖掘利好因素 |
| **空头研究员** | 寻找做空/回避理由 | 论证下跌风险、挖掘利空因素 |

### Phase 3: 风险分析师（三派辩论）

| Agent | 风险偏好 | 典型观点 |
|-------|----------|----------|
| **激进分析师** | 高风险高收益 | 重仓出击，把握成长机会 |
| **中性分析师** | 平衡风险收益 | 分批建仓，控制仓位 |
| **保守分析师** | 资本保护优先 | 等待更好价格，严格止损 |

### Phase 4-6: 决策层（串行执行）

| Agent | 职责 | 输出 |
|-------|------|------|
| **风险经理** | 综合评估风险，制定风控措施 | 风控决策报告、仓位上限、止损纪律 |
| **研究经理** | 汇总分析结果，制定投资计划 | 投资计划书、目标价位、持有周期 |
| **交易员** | 最终交易决策，执行细节 | 最终决策报告、具体操作建议 |

## 使用方法

### 在 Claude Code 中使用

```bash
# 分析港股（使用 .HK 后缀）
/stock-analysis 分析 00700.HK

# 分析 A 股（直接使用股票代码）
/stock-analysis 分析 600519

# 使用中文名称
/stock-analysis 分析 泡泡玛特
/stock-analysis 分析 茅台
```

### 输出报告

所有分析报告保存在 `/tmp/stock-analysis/{股票代码}/reports/` 目录下：

```
/tmp/stock-analysis/09992.HK/reports/
├── technical.md          # 技术分析报告
├── news.md               # 新闻分析报告
├── market.md             # 市场分析报告
├── fundamentals.md       # 基本面分析报告
├── sentiment.md          # 情绪分析报告
├── china.md              # 中国市场分析报告
├── social_media.md       # 社交媒体分析报告
├── bull.md               # 多头观点
├── bear.md               # 空头观点
├── risk_aggressive.md    # 激进风险分析
├── risk_neutral.md       # 中性风险分析
├── risk_conservative.md  # 保守风险分析
├── risk_manager.md       # 风险经理决策
├── research_manager.md   # 研究经理投资计划
└── final_trader.md       # 最终交易决策
```

## 最终报告内容

`final_trader.md` 包含完整的投资决策：

1. **综合研判** - 多维度分析汇总、核心矛盾识别
2. **交易决策** - 买入/持有/卖出建议、置信度评分
3. **交易参数** - 入场价、目标价、止损价、建议仓位
4. **执行计划** - 分阶段建仓/止盈/止损策略
5. **风险提示** - 核心风险清单与应对措施
6. **后续跟踪** - 定期复盘计划、关键事件监控

## 技术特点

### 1. 并行执行提升效率

Phase 1 的 7 个分析师同时运行，大幅缩短分析时间。

### 2. 辩论机制保证客观性

多空对抗、三派辩论确保从不同角度审视投资机会，避免单一视角盲点。

### 3. 专业分工

每个 Agent 都有专门设计的 prompt，确保分析深度和专业性。

### 4. 完整决策链

从信息收集 → 分析 → 辩论 → 风险评估 → 最终决策，完整模拟专业投资机构的决策流程。
